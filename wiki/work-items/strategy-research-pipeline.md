# Strategy Research Pipeline

## Goal

Capture the user's two-stage strategy workflow as durable project operating memory.

## Context

The user provided a pipeline separating scheduled candidate generation from constrained build/test work, with explicit AI boundaries and trust gates before any real capital exposure.

## Decisions

- Add the pipeline to `CLAUDE.md` as standing behavior for future strategy work.
- Create a concept page so the pipeline is visible in Obsidian and linked from the index.
- Preserve the distinction between hypothesis generation and strategy approval.
- Treat portfolio construction and final capital allocation as human-operated decisions.

## Outcome

Future strategy work should produce ranked hypotheses first, prototype only filtered candidates, constrain coding agents to existing conventions, and require independent review, manual review, and secondary-account observation before live deployment.
